# v3 Topic 분류 체계 재설계 + Golden Set 생성

날짜: 2026-03-02

## 배경

v2 Topic 정확도 69.3%에서 정체. 원인 분석 결과:
- 모델 문제가 아닌 **분류 기준 자체의 모호성**
- "콘텐츠 반응 vs 투자 이야기 vs 커뮤니티 소통" 경계가 구조적으로 겹침
- "커뮤니티 소통"이 전체 33%(841건/주)로 catch-all — 내부 50% 투자, 30% 콘텐츠, 20% 인사 → **훈련 데이터 오염**

## 결정

### 1. 소재 기반 → 5분류 체계 전환

**2단계 판별 질문:**
1. "이 글에 누가 대응해야 하나?" → 운영 피드백 / 서비스 피드백
2. "글의 주된 대상이 무엇인가?" → 콘텐츠 반응 / 투자 담론 / 기타

| v2 (소재 기반) | v3 5분류 | 전환 근거 |
|-------------|---------|----------|
| 콘텐츠 반응 | **콘텐츠 반응** | 마스터/콘텐츠 대상 글 유지 |
| 투자 이야기 | **투자 담론** | 시장/투자 대상 글 유지 |
| 서비스 이슈 | **운영 피드백** / **서비스 피드백** | 사람 처리 vs 시스템 수정 분리 |
| 커뮤니티 소통 | 콘텐츠 반응 33% + 투자 담론 24% + **기타** 39% | catch-all 해체 |

**왜 4분류(여론 통합)를 거부했나:**
- 초기 설계에서 콘텐츠 반응 + 투자 이야기 + 커뮤니티 소통을 "여론"으로 통합
- 4가지 분석 질문(마스터별 콘텐츠 감정, 증시 여론, 서비스 요구사항, 콘텐츠 불편사항) 중 3개가 Topic 필터만으로 불가능
- "여론" 안에서 카테고리 태그로 구분하는 방안도 검토했으나, 태그가 너무 많고 직관적이지 않다고 판단

### 2. 검증 모델로 Opus 4.6 선택

- 검증(golden set)은 **정확도가 제일 중요**, 비용이 아님
- 이미 Claude Code 세션에서 Opus 4.6 사용 중 → 추가 비용 $0
- 가장 강력한 모델의 결과 = 가장 신뢰할 수 있는 기준

### 3. 라벨링 파이프라인: Opus → EXAONE → KcBERT

```
Opus 4.6 golden set (102건) → EXAONE 6K건 라벨링 (Colab) → KcBERT 파인튜닝 (Colab)
```

총 비용 $0. API 없이 Colab 무료 T4로 완결.
