# Proposal: Phase 3 — 편지글/게시글 분류 체계 v5 재설계

## Intent

현행 v4 3분류(대응 필요/콘텐츠·투자/노이즈)가 사실상 **필터링**에 가까워 "데이터 라벨링"이라 부르기 어려운 상태. 콘텐츠·투자가 전체의 78%를 차지하며, 세부 구분은 LLM 자유 태그에 의존. 분류 체계를 세분화하여 태그 없이도 의미 있는 분석이 가능한 구조로 전환.

### 현행 문제

1. **3분류가 너무 넓음** — "콘텐츠·투자" 1,367건 안에 마스터 비판, 종목 분석, 강의 후기, 시장 전망이 전부 섞여있음
2. **감정 축의 필요성 불명확** — 5분류로 세분화되면 태그가 감정을 대체할 수 있음
3. **자유 태그 의존** — 통제된 카테고리 없이 LLM이 매번 태그를 생성 → 일관성 부족

### v4에서 3분류로 간 이유 (재검토)

- 콘텐츠↔투자 혼동 61% → **하지만 이건 경계 설계 문제였지 분류 수 문제가 아니었음**
- 불균형(87.9%) → **파인튜닝에서만 문제. LLM 프롬프팅에서는 불균형 무관**
- 기타/일상 경계 모호 → **"액션 불필요"로 통합했지만, 노이즈 필터링이지 분류가 아님**

## 제안: 5분류 (액션+대상 기반)

```
1. 운영 대응     — CS/운영팀이 1:1 처리 (환불, 해지, 결제, 세미나 문의, 접속 오류)
2. 서비스 피드백  — 제품팀이 수집 (앱 오류, 기능 요청, UX 문제, 콘텐츠 접근 문제)
3. 마스터 반응   — 마스터에 대한 평가/피드백 (칭찬, 비판, 건강 걱정, 신뢰/불신)
4. 시장·투자     — 종목, 포트폴리오, 시장 전망, 매매, 수익/손실
5. 일상          — 인사, 안부, 일상 잡담 (액션 불필요)
```

### v3 5분류와의 차이

| v3 | v5 (제안) | 변경 이유 |
|----|-----------|----------|
| 콘텐츠 반응 | **마스터 반응** | 콘텐츠보다 마스터가 주어인 경우가 압도적 (태그: 마스터 비판 32건, 마스터 신뢰 16건, 담쌤 32건) |
| 투자 담론 | **시장·투자** | 동일 의도 |
| 운영 피드백 | **운영 대응** | 동일 의도 |
| 서비스 피드백 | **서비스 피드백** | 동일 |
| 기타 | **일상** | "기타"보다 명확한 정의 |

### 경계 규칙 (v4에서 배운 것 적용)

- **마스터 반응 vs 시장·투자**: 마스터/강의/콘텐츠가 주어면 마스터 반응, 시장/종목/포트폴리오가 주어면 시장·투자
- **운영 대응 vs 서비스 피드백**: 1:1 처리 필요(환불, 해지) → 운영 대응, 제품 개선 사안(오류, 기능) → 서비스 피드백
- **일상**: 투자/콘텐츠/서비스 신호가 0개일 때만
- **애매하면 → 시장·투자** (기본값, v4와 동일 원칙)

### 감정 축 재검토

5분류로 세분화되면:
- 마스터 반응 안에서 비판/신뢰가 태그로 구분됨
- 시장·투자 안에서 손실/수익이 태그로 구분됨
- 운영 대응은 대부분 부정/중립
- → **별도 감정 축 없이 세부 태그로 대체 가능한지 검증 필요**

## 검증 계획

1. **골든셋 재라벨링**: 기존 227건을 v5 5분류로 재라벨링
2. **경계 케이스 분석**: voc_labelled 데이터에서 마스터 반응↔시장·투자 경계 케이스 수 확인
3. **Haiku 프롬프트 PoC**: 50건 샘플로 5분류 정확도 테스트
4. **감정 축 필요성 검증**: 감정 없이 주간 리포트 핵심 이슈가 충분히 도출되는지 확인
