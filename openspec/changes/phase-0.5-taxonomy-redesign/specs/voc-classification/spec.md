# Delta for VOC Classification — Phase 0.5: 분류 체계 재설계

## 배경

기존 Topic 4분류(콘텐츠 반응/투자 이야기/서비스 이슈/커뮤니티 소통)는 **"글의 소재"** 기준.
금융 커뮤니티 글은 본질적으로 여러 소재가 섞이므로(마스터 감사 + 투자 성과 + 인사), 소재 기반 단일 라벨은 경계가 모호.
→ Topic 정확도 69.3%에서 정체.

추가 발견: v2 "커뮤니티 소통"이 전체 33%를 차지하는 catch-all 카테고리로, 투자 50% + 콘텐츠 30% + 인사 20%가 혼재 → 훈련 데이터 오염.

## 핵심 변경

분류 기준을 **"글의 소재"(what) → "대응 주체 + 분석 대상"(who/what)** 으로 전환.

> 1단계: "이 글에 누가 대응해야 하나?"
> - 운영팀 → 운영 피드백
> - 개발팀 → 서비스 피드백
>
> 2단계: "글의 주된 대상이 무엇인가?"
> - 마스터/콘텐츠 → 콘텐츠 반응
> - 시장/종목/투자전략 → 투자 담론
> - 인사/잡담 → 기타

## MODIFIED Requirements

### Requirement: Topic 분류 체계 (수정)

기존 소재 기반 4분류를 **5분류**로 재설계한다.

#### Topic (신규)

| 값 | 분류 기준 | 예시 |
|----|----------|------|
| 운영 피드백 | 운영팀/사업팀이 사람 대 사람으로 처리할 요청·이슈 | 세미나 참석 문의, 환불 요청, 멤버십 문의, 배송 문의 |
| 서비스 피드백 | 개발팀이 시스템을 수정해야 하는 기술적 이슈·요청 | 앱 크래시, 결제 오류(시스템), 로그인 실패, 기능 요청, 사칭 제보 |
| 콘텐츠 반응 | 마스터/콘텐츠가 주된 대상인 감상·반응·피드백 | 강의 후기, 마스터 칭찬/불만, 리포트 반응, 콘텐츠 품질 의견 |
| 투자 담론 | 시장/종목/투자전략이 주된 대상인 의견·질문·공유 | 투자 질문, 수익/손실 공유, 시장 분석, 종목 토론 |
| 기타 | 분석 가치가 낮은 인사·잡담·테스트 | 새해 인사, 안부, 축하, 테스트 글 |

#### Topic (폐기)

| 값 | 폐기 사유 |
|----|----------|
| ~~콘텐츠 반응~~ (v2) | "투자 이야기"와 경계 모호 (마스터 감사 + 투자 성과 혼재) |
| ~~투자 이야기~~ (v2) | "콘텐츠 반응"과 경계 모호 |
| ~~서비스 이슈~~ (v2) | 운영(사람 처리) vs 서비스(개발 처리) 구분 필요 |
| ~~커뮤니티 소통~~ (v2) | catch-all 카테고리 — 33%가 이 카테고리, 내부 61%가 잘못 라벨 |
| ~~여론~~ (v3 초기안) | "콘텐츠 반응 + 투자 이야기 + 커뮤니티 소통" 통합 → 분석 자유도 저하 |

#### 경계 판별 규칙

| 글 | 분류 | 판별 근거 |
|----|------|----------|
| "결제가 안 돼요" | 서비스 피드백 | 시스템 장애 → 개발팀 |
| "환불해주세요" | 운영 피드백 | 정책/프로세스 → 운영팀 |
| "결제 정책이 불합리해요" | 운영 피드백 | 정책 이슈 → 사업팀 |
| "쌤 강의 너무 좋아요 덕분에 수익 났어요" | 콘텐츠 반응 | 주된 대상 = 마스터/강의 |
| "쌤 리포트대로 매수했는데 수익 30% 났어요" | 콘텐츠 반응 | 마스터 리포트에 대한 반응 |
| "삼성전자 지금 들어가도 될까요?" | 투자 담론 | 주된 대상 = 종목/매매 |
| "요즘 장이 불안한데 현금 비중 어떻게 가져가세요?" | 투자 담론 | 주된 대상 = 시장/투자전략 |
| "ARK 그래프가 옛날 거라 답답해요" | 콘텐츠 반응 | 콘텐츠 품질 불만 |
| "앱에서 강의가 안 보여요" | 서비스 피드백 | 기술 이슈 → 개발팀 |
| "새해 복 많이 받으세요" | 기타 | 인사 |

#### Scenario: Topic 분류 정확도 목표
- GIVEN 신규 Topic 5분류 체계
- WHEN 동일 테스트셋으로 벤치마크
- THEN Topic 정확도 80%+ 달성 (기존 69.3% 대비 +10%p 이상)
- AND LLM의 분류 일관성 90%+ (동일 건 재분류 시 동일 결과)

#### Scenario: 활용도 시뮬레이션 (완료)
- GIVEN golden set 102건의 v3 5분류 결과
- WHEN 7개 분석 시나리오(analysis-scenarios.md)를 v3 기준으로 실행
- THEN 각 시나리오의 쿼리가 의미 있는 결과를 반환함
- AND v2에서 가능했던 분석이 v3에서도 동등 이상으로 가능함
- **결과**: 4개 분석 질문(마스터별 콘텐츠 감정, 증시 여론, 서비스 요구사항, 콘텐츠 불편사항) 모두 Topic 필터만으로 대응 가능

### Requirement: 검증 프로세스 (신규)

신규 분류 체계의 명확성을 검증한 후 파인튜닝에 사용할 훈련 데이터를 생성한다.

#### Scenario: Golden Set 생성 (완료)
- GIVEN 기존 분류된 데이터에서 v2 Topic별 균형 추출 52건 + 주간 데이터 추가 샘플 50건
- WHEN Opus 4.6이 v3 5분류 체계로 분류
- THEN `data/gold_dataset/v5_golden_set.json`에 저장됨 (102건)
- AND 평균 신뢰도 ~0.87, border cases 10건 (9.8%)
- **결과**: 콘텐츠 반응 34, 투자 담론 28, 기타 23, 운영 피드백 14, 서비스 피드백 3

#### Scenario: 활용도 시뮬레이션 (완료)
- GIVEN golden set의 v3 5분류 결과
- WHEN 4개 구체적 분석 질문으로 테스트
- THEN Topic 필터만으로 모든 질문에 대응 가능 확인됨
- AND 초기 4분류("여론" 통합)에서 발견된 분석 자유도 부족 문제 해결됨

#### Scenario: 전체 데이터 라벨링
- GIVEN 활용도 시뮬레이션 통과 + 분류 체계 확정
- WHEN EXAONE-3.5-7.8B로 6K건 전체 라벨링 (Colab T4)
- THEN 라벨링 결과가 `data/training_data/v3/labeled_all.json`에 저장됨
- AND 이 데이터로 KcBERT 재학습 가능

#### Scenario: KcBERT 파인튜닝
- GIVEN EXAONE 라벨링 완료 데이터
- WHEN train/val/test split 후 klue/roberta-base 파인튜닝 (Colab T4)
- THEN `models/v3/topic/final_model/`에 모델 저장됨
- AND golden set 기준 정확도 80%+ 달성 (v2 69.3% 대비)

## 기존 체계와의 관계

- v2 코드(classifier_v2/)는 변경하지 않음
- v3(classifier_v3/)에서 신규 체계 구현
- 기존 리포트 파이프라인은 그대로 유지, 신규 파이프라인 별도 구축
- 기존 분류 데이터(`data/classified_data_two_axis/`)는 보존
