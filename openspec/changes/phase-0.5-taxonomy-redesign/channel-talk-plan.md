# 채널톡 CS 데이터 분류 — 논의 현황

## 데이터 탐색 결과 (완료)

### 데이터 범위
- **소스**: BigQuery `us-service-data.channel_io.messages`
- **기간**: 2024-08-06 ~ 2025-12-05 (약 16개월)
- **총 메시지**: 431,589건
- **2025-07 이후 급증**: 월 53K~85K건 (그전은 월 1K~18K건)

### 데이터 구조
- `chatId`별 대화 단위, 주간 ~1,000건 대화
- `personType` 분포: bot 68% / user 20% / manager 12%
- 대화당 평균 22건 메시지, 사용자 메시지 평균 4.5건
- `createdAt`: Unix timestamp (ms), INTEGER 타입

### 주요 키워드 (2025-11 기준)
환불, 취소, 구독, 해지, 멤버십, 결제, 패키지, 겨울학기, 자료, 계좌이체
→ **전형적인 CS 문의 데이터** (커뮤니티 VOC와 완전히 다른 성격)

## 분류 목적 (확정)

1. **서비스·콘텐츠 개선**: 어떤 불편/요구가 많은지 파악
2. **CS 개발 지원 판단**: 자동화·봇 개선이 필요한 영역 식별
3. **수강 데이터 조인** (향후): 강의 런칭 후 문의 패턴, 환불 사유 분석
4. **시계열 통계**: 날짜·마스터별 문의 분포 추적

## 분류 체계 (안) — 추가 논의 필요

### 1차 제안: 5분류

| Topic | 설명 | 예시 |
|-------|------|------|
| 결제·환불 | 결제, 취소, 환불, 계좌이체 | "결제 취소했는데 환불 안됨" |
| 구독·멤버십 | 구독, 해지, 변경, 갱신 | "멤버십 해지하고 싶어요" |
| 콘텐츠·수강 | 강의 접근, 자료, 수강 문의 | "노트북에서 듣고싶어요" |
| 서비스·기술 | 로그인, 앱 오류, 기능 문의 | "커뮤니티가 뭔가요" |
| 기타 | 위에 해당 안 되는 문의 | 단순 인사 등 |

### 미결 사항

- [ ] 분류 체계 확정 (5분류가 적절한지, 세분화 필요한지)
- [ ] `결제·환불`과 `구독·멤버십` 경계 — 해지 시 환불 문의는 어디?
- [ ] 수강 데이터 조인 키 확인 (personId → userId 매핑 가능 여부)
- [ ] bot 응답 참고 여부 — 사용자 메시지만 분류? 대화 맥락 포함?
- [ ] 마스터 정보 — chatId에서 마스터 매핑 가능한지 (별도 테이블 필요?)
- [ ] 분류 파이프라인 — v3와 동일 (Opus 라벨링 → KcBERT 파인튜닝)?
