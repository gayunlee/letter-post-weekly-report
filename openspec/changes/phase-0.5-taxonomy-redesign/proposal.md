# Proposal: Phase 0.5 — Topic 분류 체계 재설계

## Intent

기존 Topic 4분류(콘텐츠 반응/투자 이야기/서비스 이슈/커뮤니티 소통)의 **소재 기반** 경계 모호성으로 인한 정확도 한계(69.3%)를 해결하기 위해, **5분류 체계**로 전환한다.

현재 문제:
1. "담쌤 덕분에 수익 35% 감사합니다" → 콘텐츠 반응? 투자 이야기? 커뮤니티 소통? (3개 겹침)
2. 금융 커뮤니티 글은 본질적으로 여러 소재가 혼재 → 소재 기반 단일 라벨은 구조적 한계
3. "커뮤니티 소통"이 전체 33%로 catch-all — 내부 61%가 잘못 라벨 → **훈련 데이터 오염**
4. "서비스 이슈"에 운영팀 처리 건과 개발팀 처리 건이 혼재 → 라우팅 불가

## 핵심 전환

분류 기준: **"이 글이 무엇에 대한 건가?" → "누가 대응하나?" + "주된 대상이 뭔가?"**

| v2 (폐기) | v3 5분류 | 전환 근거 |
|-----------|---------|----------|
| 콘텐츠 반응 | **콘텐츠 반응** | 마스터/콘텐츠 대상 유지 (대상 기반 재정의) |
| 투자 이야기 | **투자 담론** | 시장/투자 대상 유지 (대상 기반 재정의) |
| 서비스 이슈 | **운영 피드백** / **서비스 피드백** | 사람 처리 vs 시스템 수정 분리 |
| 커뮤니티 소통 | 콘텐츠 반응/투자 담론/**기타** | catch-all 해체 |

**왜 4분류(여론 통합)를 거부했나:**
- 초기 "여론" 통합 시 4개 분석 질문 중 3개가 Topic 필터만으로 불가능
- 태그 기반 하위 구분 방안도 "태그가 너무 많고 직관적이지 않다"고 판단
