# VOC 데이터 분석 시나리오 — 부서별 활용 가이드

> **v3 5분류 체계 반영** (2026-03-02)
> Topic: 운영 피드백 / 서비스 피드백 / 콘텐츠 반응 / 투자 담론 / 기타

## 분류 체계 요약

모든 편지글/게시글은 다음 필드를 가집니다:

| 필드 | 설명 | 예시 |
|------|------|------|
| **주제(Topic)** | 운영 피드백 / 서비스 피드백 / 콘텐츠 반응 / 투자 담론 / 기타 | "콘텐츠 반응" |
| **감성(Sentiment)** | 긍정 / 부정 / 중립 | "부정" |
| **카테고리 태그** | 29개 controlled vocabulary (주제별 4~7개) | "결제/환불 정책" |
| **자유 태그** | 구체적 명사구 2~3개 | "이중 결제", "iOS 결제 오류" |
| **요약** | 15~40자 한줄 요약 | "iOS 앱 내 이중 결제 발생 제보" |
| **의도(Intent)** | 질문/요청 / 피드백/의견 / 제보/건의 / 정보공유 | "제보/건의" |
| **마스터/클럽** | 어떤 마스터의 커뮤니티인지 | "미과장 / 투자상사" |
| **날짜** | 작성 시점 | "2026-02-15" |

### v3 Topic 분류 기준

| Topic | 분류 기준 | 대응 주체 |
|-------|----------|----------|
| **운영 피드백** | 사람 대 사람으로 처리할 요청·이슈 | 운영팀/사업팀 |
| **서비스 피드백** | 시스템 수정이 필요한 기술적 이슈·요청 | 개발팀 |
| **콘텐츠 반응** | 마스터/콘텐츠가 주된 대상인 감상·피드백 | 콘텐츠팀/사업팀 (모니터링) |
| **투자 담론** | 시장/종목/투자전략이 주된 대상 | 없음 (모니터링) |
| **기타** | 분석 가치 낮은 인사·잡담 | 없음 |

## 시나리오 1: CS/운영팀 — "지금 당장 대응해야 할 문제가 뭐야?"

### 질문 예시
> "이번 주 운영 피드백 중 부정 의견 보여줘"

### 쿼리 조건
```
주제 = "운영 피드백"
감성 = "부정"
```

### 기대 결과
| 카테고리 태그 | 건수 | 대표 사례 (요약) |
|---|---|---|
| 결제/환불 정책 | 5 | "환불 처리 2주째 미완료 문의" |
| 구독/멤버십 관리 | 3 | "멤버십 해지 방법 안내 부족" |
| 가격/프로모션 정책 | 2 | "기존 대비 가격 인상 부담" |

### 활용
- 운영팀이 자기 소관 건만 정확히 파악 (기술 이슈는 "서비스 피드백"에서 별도 확인)
- 환불/멤버십/배송 등 카테고리 태그로 세부 분류
- 주간 단위 추이로 이슈 해결 확인

## 시나리오 2: 개발팀 — "기술적으로 고쳐야 할 게 뭐야?"

### 질문 예시
> "이번 주 서비스 피드백 전체 보여줘. 특히 결제 시스템이나 앱 오류 건"

### 쿼리 조건
```
주제 = "서비스 피드백"
카테고리 태그 IN ("앱/기능 오류", "결제 시스템 오류", "로그인/계정 문제")
```

### 기대 결과
| 카테고리 태그 | 건수 | 감성 | 대표 사례 |
|---|---|---|---|
| 앱/기능 오류 | 4 | 부정 3, 중립 1 | "앱 종료 반복, 게시글 로딩 실패" |
| 결제 시스템 오류 | 2 | 부정 2 | "iOS 업데이트 후 구독 결제 이중 청구" |
| 기능/UX 개선 요청 | 3 | 중립 3 | "다크모드 지원 요청" |

### 활용
- 개발팀이 시스템 수정 건만 정확히 파악 (운영 처리 건은 "운영 피드백"에서 별도)
- 반복 제보 패턴으로 장애 우선순위 결정
- 기능 요청은 백로그에 추가

## 시나리오 3: 사업팀 — "내가 맡은 마스터 커뮤니티 분위기는 어때?"

### 질문 예시
> "미과장 클럽의 이번 주 부정 의견 요약해줘. 지난주 대비 어때?"

### 쿼리 조건
```
마스터 = "미과장"
감성 = "부정"
기간 = 이번 주 vs 전주
```

### 기대 결과
```
[미과장 / 투자상사] 이번 주 부정 비율: 18% (전주 12%, +6%p)

주요 부정 의견:
1. 콘텐츠 반응·부정 (5건)
   - 카테고리 태그: 콘텐츠 품질/깊이 (3건), 마스터 소통/태도 (2건)
   - "최근 브리핑 내용이 반복적" (×3)
   - "포트폴리오 변경 사전 설명 부족" (×2)
2. 투자 담론·부정 (3건)
   - 카테고리 태그: 수익/손실 공유 (2건), 포트폴리오/종목 전략 (1건)
   - "추천 종목 손실 발생"
3. 운영 피드백·부정 (2건)
   - 카테고리 태그: 가격/프로모션 정책 (2건)
   - "가격 인상 대비 콘텐츠 질 하락"
```

### 활용
- 마스터별 부정 비율 추이를 추적하여 관리 포인트 식별
- 부정 급증 시 마스터에게 피드백 전달
- 콘텐츠 불만 vs 투자 성과 불만을 Topic으로 즉시 구분

## 시나리오 4: 콘텐츠팀 — "콘텐츠 만족도와 개선 포인트는?"

### 질문 예시
> "이번 주 콘텐츠 반응 중 부정 의견 패턴 분석해줘"

### 쿼리 조건
```
주제 = "콘텐츠 반응"
감성 = "부정"
```

### 기대 결과
```
콘텐츠 반응 부정 (38건):

카테고리 태그 분포:
  콘텐츠 품질/깊이: 15건 (39%)
  마스터 소통/태도: 9건 (24%)
  리포트/브리핑 반응: 8건 (21%)
  강의/수업 반응: 6건 (16%)

주요 자유 태그:
  "리포트 업데이트 빈도 저하" (5건)
  "과거 데이터 재활용" (4건)
  "실시간 방송 시간대 불편" (3건)
```

### v2 대비 개선
- v2: "콘텐츠 반응 + 부정" 필터 → 투자 성과 공유 글이 섞일 수 있었음
- **v3**: "콘텐츠 반응" Topic이 마스터/콘텐츠 대상 글만 포함 → 태그 없이 Topic만으로 필터 가능
- 투자 관련 불만(종목 손실 등)은 "투자 담론"에 분리되어 노이즈 제거

### 활용
- 콘텐츠 품질 개선 포인트 도출
- 마스터별 콘텐츠 피드백을 직접 전달
- 새로운 콘텐츠 기획 아이디어 수집

## 시나리오 5: 사업팀 — "신규 상품 런칭 후 반응이 어때?"

### 질문 예시
> "2월 10일 런칭한 '미과장 프리미엄 멤버십' 관련 반응 2주치 보여줘"

### 쿼리 조건
```
마스터 = "미과장"
기간 = 2026-02-10 ~ 2026-02-24
자유 태그 CONTAINS ("멤버십", "프리미엄", "구독", "가격")
OR 카테고리 태그 IN ("가격/프로모션 정책", "구독/멤버십 관리")
```

### 기대 결과
```
'프리미엄 멤버십' 관련 반응 (23건):

Topic 분포:
  콘텐츠 반응 10건 (43%) — 프리미엄 콘텐츠 품질 감상
  운영 피드백 8건 (35%) — 구독 문의, 환불 요청
  투자 담론 3건 (13%) — 프리미엄 전략 관련 질문
  기타 2건 (9%) — 축하/응원

감성 분포: 긍정 9건(39%) / 중립 8건(35%) / 부정 6건(26%)

긍정 키워드: "프리미엄 리포트 퀄리티 만족", "실시간 소통 좋다"
부정 키워드: "기존 대비 가격 인상 부담", "무료 콘텐츠 축소 불만"
```

### 활용
- 상품 런칭 반응이 5개 Topic으로 자동 구조화
- 운영팀은 "운영 피드백 8건"만 처리, 콘텐츠팀은 "콘텐츠 반응 10건" 모니터링

## 시나리오 6: 전사 — "증시 상황에 따른 이용자 감정 변화는?"

### 질문 예시
> "지난 4주간 투자 담론의 감성 추이 보여줘"

### 쿼리 조건
```
주제 = "투자 담론"
기간 = 최근 4주 (주차별 집계)
```

### 기대 결과
```
투자 담론 감성 추이 (4주):

| 주차 | 전체 | 긍정 | 부정 | 중립 | 부정 비율 |
|------|------|------|------|------|----------|
| 1/27~2/2 | 180 | 45% | 20% | 35% | 20% |
| 2/3~2/9 | 210 | 30% | 35% | 35% | 35% ↑ |
| 2/10~2/16 | 195 | 25% | 42% | 33% | 42% ↑ |
| 2/17~2/23 | 220 | 48% | 18% | 34% | 18% ↓ |

2/3~2/16 부정 급증 → KOSPI 하락 구간과 일치
```

### v2 대비 개선
- v2: "투자 이야기" 필터 → 마스터 감사글, 인사 등이 섞여 증시 감정 왜곡
- **v3**: "투자 담론"이 시장/종목/투자전략 대상 글만 포함 → 증시 감정 지표로서 정확도 향상
- 카테고리 태그로 세부 분석: "시장 전망/매크로" vs "포트폴리오/종목 전략"

### 활용
- 증시 하락 시 이용자 감정 변화 모니터링
- 부정 급증 → 이탈 위험 감지 → 운영 대응 트리거
- 마스터별 투자 담론 감성으로 커뮤니티 건강도 평가

## 시나리오 7: 전사 — "이번 주 전체 VOC 핵심 요약"

### 질문 예시
> "이번 주 전체 편지글/게시글 핵심 이슈 3줄로 요약해줘"

### 기대 결과
```
1. 서비스 피드백: 앱/기능 오류 4건 + 결제 시스템 오류 2건
   → iOS 업데이트 이후 이중 결제 이슈 집중, 개발팀 확인 필요
2. 운영 피드백: 환불/멤버십 관련 문의 8건 (전주 대비 +3건)
   → 프리미엄 멤버십 런칭 후 가격 정책 질문 증가
3. 콘텐츠 반응: 미과장 클럽 콘텐츠 품질 불만 7건 (전주 2건)
   → 리포트 업데이트 빈도 하락 지적, 마스터 피드백 필요
```

### 활용
- 요약이 **Topic별로 구조화** → 각 팀이 자기 소관 항목을 즉시 파악
- 서비스 피드백 → 개발팀, 운영 피드백 → 운영팀, 콘텐츠 반응 → 콘텐츠팀

## 핵심 비교: v2 vs v3 시나리오 대응력

| 시나리오 | v2 대응 | v3 대응 | 개선 |
|---------|--------|--------|------|
| CS 긴급 대응 | "서비스 이슈+부정" → 운영 건 혼재 | "서비스 피드백" → 기술 건만 | 노이즈 제거 |
| 운영 처리 건 | "서비스 이슈"에서 수동 분리 | "운영 피드백" → 자동 분리 | 신규 가능 |
| 콘텐츠 피드백 | "콘텐츠 반응+부정" → 투자글 혼재 | "콘텐츠 반응" → 마스터/콘텐츠 건만 | **Topic 필터만으로 가능** |
| 투자 여론 | "투자 이야기" → 인사/잡담 혼재 | "투자 담론" → 시장/투자 건만 | **증시 감정 지표화** |
| 마스터별 분석 | Topic 3개 합산 필요 | Topic별 구조화된 내역 | 즉시 원인 파악 |
| 부정 급증 감지 | Sentiment 축 → 동일 | Sentiment 축 → 동일 | 변화 없음 |
| 전사 요약 | 소재별 나열 | 대응 주체/대상별 구조화 | 행동 가능성 향상 |

## 채팅 인터페이스 구상

위 시나리오들을 **자연어 채팅**으로 질문할 수 있는 웹 인터페이스를 구축합니다.

### 구조
```
[사용자 질문] → LLM이 쿼리 조건 추출 → 분류 데이터 필터링 → LLM이 인사이트 요약 → [답변]
                                                                         ↓
                                                              [원본 데이터 링크]
```

### 핵심 기능
1. **자연어 쿼리**: "이번 주 결제 관련 불만 몇 건이야?" → Topic=서비스 피드백 OR 운영 피드백, Sentiment=부정, category_tag LIKE 결제%
2. **원본 확인**: 답변과 함께 원문 데이터 테이블 제공 (검증 경로)
3. **기간 비교**: "지난주 대비" 같은 시간 비교 자동 처리
4. **마스터/클럽 필터**: "미과장 클럽만" 같은 범위 지정
5. **추이 시각화**: 주차별 감성 추이 차트 자동 생성

### 검증 경로
- 모든 답변에 **근거 데이터 건수** + **원본 보기 링크** 제공
- 사용자가 원본을 직접 확인하여 분류 정확도를 체감할 수 있음
- 오분류 발견 시 피드백 → 검수 데이터로 축적 → 모델 개선 사이클
