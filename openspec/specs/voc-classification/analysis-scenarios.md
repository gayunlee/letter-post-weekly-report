# VOC 데이터 분석 시나리오 — 부서별 활용 가이드

## 분류 체계 요약

모든 편지글/게시글은 다음 필드를 가집니다:

| 필드 | 설명 | 예시 |
|------|------|------|
| **주제(Topic)** | 콘텐츠 반응 / 투자 이야기 / 서비스 이슈 / 커뮤니티 소통 | "서비스 이슈" |
| **감성(Sentiment)** | 긍정 / 부정 / 중립 | "부정" |
| **카테고리 태그** | 28개 controlled vocabulary (주제별 6~8개) | "결제/환불/구독" |
| **자유 태그** | 구체적 명사구 2~3개 | "이중 결제", "iOS 결제 오류" |
| **요약** | 15~40자 한줄 요약 | "iOS 앱 내 이중 결제 발생 제보" |
| **의도(Intent)** | 질문/요청 / 피드백/의견 / 제보/건의 / 정보공유 | "제보/건의" |
| **마스터/클럽** | 어떤 마스터의 커뮤니티인지 | "미과장 / 투자상사" |
| **날짜** | 작성 시점 | "2026-02-15" |

## 시나리오 1: CS팀 — "지금 당장 대응해야 할 문제가 뭐야?"

### 질문 예시
> "이번 주 서비스 관련 불만 사항 중 결제나 로그인 관련 건 보여줘"

### 쿼리 조건
```
주제 = "서비스 이슈"
감성 = "부정"
카테고리 태그 IN ("결제/환불/구독", "앱/기능 오류", "콘텐츠 접근 문제")
```

### 기대 결과
| 카테고리 태그 | 건수 | 대표 사례 (요약) |
|---|---|---|
| 결제/환불/구독 | 12 | "iOS 업데이트 후 구독 결제 이중 청구" |
| 앱/기능 오류 | 8 | "앱 종료 반복, 게시글 로딩 실패" |
| 콘텐츠 접근 문제 | 3 | "유료 리포트 열람 불가 오류" |

### 활용
- 반복 제보되는 결제/로그인 이슈를 즉시 파악
- 원문 확인 후 장애 대응 여부 판단
- 주간 단위 추이로 이슈 해결 확인

## 시나리오 2: 사업팀 — "내가 맡은 마스터 커뮤니티 분위기는 어때?"

### 질문 예시
> "미과장 클럽의 이번 주 부정 여론 요약해줘. 지난주 대비 어때?"

### 쿼리 조건
```
마스터 = "미과장"
감성 = "부정"
기간 = 이번 주 vs 전주
```

### 기대 결과
```
[미과장 / 투자상사] 이번 주 부정 비율: 18% (전주 12%, +6%p)

주요 부정 의견:
1. 콘텐츠 반응·부정 (7건)
   - "최근 브리핑 내용이 반복적" (×3)
   - "포트폴리오 변경 사전 설명 부족" (×2)
2. 투자 이야기·부정 (4건)
   - "추천 종목 손실 지속"
   - "시장 하락기 대응 가이드 없음"
```

### 활용
- 마스터별 부정 비율 추이를 추적하여 관리 포인트 식별
- 부정 급증 시 마스터에게 피드백 전달
- 주간 리포트에서 마스터별 핵심 이슈 브리핑

---

## 시나리오 3: 콘텐츠팀 — "콘텐츠 만족도와 개선 포인트는?"

### 질문 예시
> "콘텐츠 반응 중 부정 의견 패턴 분석해줘. 어떤 콘텐츠가 불만이 많아?"

### 쿼리 조건
```
주제 = "콘텐츠 반응"
감성 = "부정"
```

### 기대 결과
```
콘텐츠 불만 패턴 (38건):

카테고리 태그 분포:
  콘텐츠 품질/깊이: 15건 (39%)
  마스터 소통/태도: 9건 (24%)
  리포트/브리핑 피드백: 8건 (21%)
  콘텐츠 주제 요청: 6건 (16%)

주요 자유 태그:
  "리포트 업데이트 빈도 저하" (5건)
  "과거 데이터 재활용" (4건)
  "실시간 방송 시간대 불편" (3건)

마스터별:
  미과장: 12건 — "브리핑 반복", "ARK 데이터 오래됨"
  김마스터: 8건 — "강의 깊이 부족", "초보 내용 위주"
```

### 활용
- 콘텐츠 품질 개선 포인트 도출
- 마스터별 콘텐츠 피드백을 직접 전달
- "콘텐츠 주제 요청" 태그로 새로운 콘텐츠 기획 아이디어 수집

## 시나리오 4: 사업팀 — "신규 상품 런칭 후 반응이 어때?"

### 질문 예시
> "2월 10일 런칭한 '미과장 프리미엄 멤버십' 관련 반응 2주치 보여줘"

### 쿼리 조건
```
마스터 = "미과장"
기간 = 2026-02-10 ~ 2026-02-24
자유 태그 CONTAINS ("멤버십", "프리미엄", "구독", "가격")
OR 카테고리 태그 IN ("가격/프로모션 정책", "결제/환불/구독")
```

### 기대 결과
```
'프리미엄 멤버십' 관련 반응 (23건):

감성 분포: 긍정 9건(39%) / 중립 8건(35%) / 부정 6건(26%)

긍정 키워드: "프리미엄 리포트 퀄리티 만족", "실시간 소통 좋다"
부정 키워드: "기존 대비 가격 인상 부담", "무료 콘텐츠 축소 불만"

주차별 추이:
  1주차(2/10~16): 긍정 60%, 부정 15% — 초기 호응
  2주차(2/17~23): 긍정 30%, 부정 35% — 가격 불만 증가
```

### 활용
- 상품 런칭 후 초기 반응 모니터링
- 가격 민감도, 콘텐츠 기대치 파악
- 2주차 부정 급증 시 가격 정책 재검토 근거

## 시나리오 5: 전사 — "증시 상황에 따른 이용자 감정 변화는?"

### 질문 예시
> "지난 4주간 투자 이야기 주제의 감성 추이 보여줘. 시장 하락기와 겹치는지 확인하고 싶어"

### 쿼리 조건
```
주제 = "투자 이야기"
기간 = 최근 4주 (주차별 집계)
```

### 기대 결과
```
투자 이야기 감성 추이 (4주):

| 주차 | 전체 | 긍정 | 부정 | 중립 | 부정 비율 |
|------|------|------|------|------|----------|
| 1/27~2/2 | 280 | 45% | 20% | 35% | 20% |
| 2/3~2/9 | 310 | 38% | 28% | 34% | 28% ↑ |
| 2/10~2/16 | 295 | 30% | 35% | 35% | 35% ↑ |
| 2/17~2/23 | 320 | 42% | 22% | 36% | 22% ↓ |

2/3~2/16 부정 급증 → KOSPI 2,500 → 2,380 하락 구간과 일치
2/17~ 부정 감소 → 시장 반등 + 마스터 시장 분석 콘텐츠 증가 효과

주요 부정 자유 태그: "손실 확대", "추천 종목 하락", "불안감"
주요 긍정 자유 태그: "반등 수익", "저가 매수 성공", "마스터 분석 적중"
```

### 활용
- 시장 상황과 이용자 감정의 상관관계 파악
- 하락기에 마스터가 어떤 콘텐츠를 제공해야 하는지 근거 제시
- 경영진 보고용 시장 영향도 데이터

## 시나리오 6: 운영팀 — "커뮤니티 건강도 점검"

### 질문 예시
> "커뮤니티 소통에서 부정 비율이 높은 클럽 top 5 알려줘"

### 쿼리 조건
```
주제 = "커뮤니티 소통"
감성 = "부정"
GROUP BY 클럽
ORDER BY 부정비율 DESC
LIMIT 5
```

### 기대 결과
```
커뮤니티 부정 비율 상위 클럽:

1. [A클럽] 부정 28% (전주 15%, +13%p ↑)
   자유 태그: "게시판 분위기 악화", "회원 간 갈등"
2. [B클럽] 부정 22% (전주 20%, +2%p)
   자유 태그: "마스터 소통 부족", "질문 무응답"
...
```

### 활용
- 커뮤니티 분위기 악화 조기 감지
- 운영 개입이 필요한 클럽 우선순위 결정

### 쿼리 조건

## 채팅 인터페이스 구상

위 시나리오들을 **자연어 채팅**으로 질문할 수 있는 웹 인터페이스를 구축합니다.

### 구조
```
[사용자 질문] → LLM이 쿼리 조건 추출 → 분류 데이터 필터링 → LLM이 인사이트 요약 → [답변]
                                                                         ↓
                                                              [원본 데이터 링크]
```

### 핵심 기능
1. **자연어 쿼리**: "이번 주 결제 관련 불만 몇 건이야?" → 자동으로 Topic=서비스이슈, Sentiment=부정, category_tag=결제/환불/구독 필터
2. **원본 확인**: 답변과 함께 원문 데이터 테이블 제공 (검증 경로)
3. **기간 비교**: "지난주 대비" 같은 시간 비교 자동 처리
4. **마스터/클럽 필터**: "미과장 클럽만" 같은 범위 지정
5. **추이 시각화**: 주차별 감성 추이 차트 자동 생성

### 검증 경로
- 모든 답변에 **근거 데이터 건수** + **원본 보기 링크** 제공
- 사용자가 원본을 직접 확인하여 분류 정확도를 체감할 수 있음
- 오분류 발견 시 피드백 → 검수 데이터로 축적 → 모델 개선 사이클

### 기술 스택 (예상)
- 프론트: Streamlit 또는 Next.js + 채팅 UI
- 백엔드: 분류 데이터 JSON/BigQuery에서 조회
- LLM: Claude (쿼리 생성 + 인사이트 요약)
- 검증: 답변마다 참조 데이터 ID 목록 제공
